## 4. Быстродействие и безопасность

**Уже хорошо:**

- Рассылка SOS: фоновые задачи + задержка 50 ms между сообщениями (Telegram rate limits).  
- `MaxAdapter`: переиспользование `aiohttp.ClientSession`.  
- Webhook: `hmac.compare_digest`, проверка IP и/или подписи тела.  
- Блокировка пользователей в middleware; SOS и юридические callback обходят блок только где нужно.

**Рекомендации без смены архитектуры:**

- За nginx: убедиться, что подпись YooKassa проверяется (ключ не пустой), либо настроить доверенный проброс IP.  
- Не ослаблять условие «подпись ИЛИ IP» без понимания последствий для продакшена.

**Не делалось:** объединение двух SQL-запросов по телефону в один `UNION` — это ломает порядок «сначала пилот с другой платформы, иначе пассажир» при конфликтующих данных.
