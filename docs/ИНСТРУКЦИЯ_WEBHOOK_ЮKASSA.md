# Настройка webhook ЮKassa для мото-бота

Простая пошаговая инструкция. Домен на reg.ru, сервер на Beget (IP: 46.173.18.7).

---

## Шаг 1. Настройка DNS на reg.ru

Чтобы домен работал с вашим сервером, нужно добавить DNS-запись.

1. Зайдите на сайт **reg.ru** и войдите в личный кабинет.
2. Откройте **«Мои домены»** и выберите нужный домен.
3. Нажмите **«Управление»** или **«DNS-серверы и управление зоной»**.
4. Найдите раздел **«Зона DNS»** или **«Добавить запись»**.
5. Добавьте новую запись:
   - **Тип записи:** A
   - **Субдомен:** оставьте пустым или `@` (если нужен корень домена)
   - **Значение / IP-адрес:** `46.173.18.7`
   - **TTL:** можно оставить по умолчанию (3600)
6. Нажмите **«Сохранить»** или **«Добавить»**.

Подождите 5–30 минут, пока изменения применятся.

---

## Шаг 2. SSL-сертификат на Beget

Webhook ЮKassa работает только по HTTPS. Нужен SSL-сертификат.

1. Зайдите в панель управления **Beget**.
2. Откройте раздел **«Сайты»**.
3. Выберите домен, который настроили на reg.ru.
4. Найдите пункт **«SSL-сертификат»** или **«Безопасность»**.
5. Включите **бесплатный сертификат Let's Encrypt** и следуйте подсказкам на экране.

Если SSL уже подключён — этот шаг можно пропустить.

---

## Шаг 3. Настройка Nginx на сервере

Нужно настроить сервер так, чтобы запросы на ваш домен перенаправлялись на бот.  
*Если у вас нет доступа по SSH — попросите настроить администратора или разработчика.*

1. Подключитесь к серверу Beget по SSH (логин и пароль есть в панели Beget → «Доступ к серверу»).
2. Откройте файл конфигурации Nginx (обычно `/etc/nginx/sites-available/default` или аналог).
3. Добавьте или отредактируйте блок (подставьте вместо `ваш-домен.ru` ваш реальный домен):

```nginx
server {
    listen 443 ssl;
    server_name ваш-домен.ru;

    ssl_certificate /path/to/fullchain.pem;
    ssl_certificate_key /path/to/privkey.pem;

    location /webhook/yookassa {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

4. Путь к сертификатам (ssl_certificate) можно посмотреть в панели Beget или в уже существующей конфигурации для вашего домена.
5. Проверьте конфигурацию и перезапустите Nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
```

6. Убедитесь, что бот запущен и слушает порт **8080**.

---

## Шаг 4. Проверка работы webhook

Перед настройкой ЮKassa проверьте, что адрес доступен:

1. Откройте в браузере: `https://ваш-домен.ru/webhook/yookassa`
2. Должна появиться надпись **OK** или пустая страница с кодом 200.

Если видите ошибку — проверьте, что бот запущен, Nginx настроен, а DNS уже обновился.

---

## Шаг 5. Настройка webhook в ЮKassa

1. Зайдите на сайт **yookassa.ru** и войдите в личный кабинет.
2. Перейдите: **Интеграция** → **HTTP-уведомления**  
   Или откройте ссылку: https://yookassa.ru/my/merchant/integration/http-notifications
3. В поле **«URL для уведомлений»** введите:
   ```
   https://ваш-домен.ru/webhook/yookassa
   ```
   Замените `ваш-домен.ru` на ваш домен.
4. В разделе событий отметьте **payment.succeeded** (успешная оплата).
5. Нажмите **«Сохранить»**.

ЮKassa может отправить тестовый запрос. Если всё настроено верно, webhook будет работать.

---

## Шаг 6. Проверка оплаты

1. Создайте тестовый платёж в боте.
2. Оплатите его (в тестовом режиме).
3. Убедитесь, что пользователю пришло уведомление об успешной оплате.
