# Роли и права

## 2.1. Таблица ролей

| Роль | Описание | Хранение |
|------|----------|----------|
| **Суперадмин** | Владелец/главный администратор. Управляет всеми городами и настройками. | Переменная окружения `SUPERADMIN_IDS` (ID через запятую) |
| **Администратор города** | Модератор конкретного города. | Таблица `city_admins` (привязка user → city) |
| **Пилот** | Мотоциклист. Создаёт/посещает мероприятия, ищет мотопару. | `User.role = pilot` |
| **Двойка** | Пассажир. Ищет пилота, создаёт/посещает мероприятия. | `User.role = passenger` |

## Конфигурация суперадминов

В `.env` задаётся список ID через запятую:

```env
# Два суперадмина (Telegram user_id)
SUPERADMIN_IDS=123456789,987654321
```

Pydantic читает `SUPERADMIN_IDS` и парсит в `list[int]`. Оба пользователя получают полный доступ к админ-панели.
